<Remember
  points={[
    <><strong>kubectl</strong> is the CLI for talking to the Kubernetes API server. It reads a config file at <code>~/.kube/config</code> (or whatever <code>$KUBECONFIG</code> points at).</>,
    <>The kubeconfig is a bundle of <strong>clusters</strong> (API endpoints), <strong>users</strong> (credentials), and <strong>contexts</strong> (named cluster+user+namespace triples).</>,
    <>Switch context with <code>kubectl config use-context &lt;name&gt;</code>. Always check <code>kubectl config current-context</code> before risky commands.</>,
  ]}
/>

<Remember
  points={[
    <><strong>Core verbs:</strong> <code>get</code> (list), <code>describe</code> (detail + events), <code>logs</code> (container output), <code>exec</code> (run inside container), <code>apply</code> (declarative create/update), <code>delete</code> (remove).</>,
    <>Debugging order: <code>describe</code> → <code>logs</code> → <code>exec</code>. Describe tells you what K8s thinks, logs tell you what the app said.</>,
    <>Use <code>kubectl logs --previous</code> (<code>-p</code>) to see logs from the last terminated instance of a crashed container.</>,
  ]}
/>

<Remember
  points={[
    <><strong>Imperative</strong> (<code>kubectl create</code>, <code>kubectl run</code>) = tell K8s what to do, now. Errors if the object already exists. Good for exploration.</>,
    <><strong>Declarative</strong> (<code>kubectl apply -f</code>) = tell K8s what state you want. Idempotent: creates or updates as needed. Standard for real environments - diffable, reviewable, reversible.</>,
    <>Prefer <code>apply</code> unless you&apos;re scratching around.</>,
  ]}
/>

<Remember
  points={[
    <>High-leverage flags: <code>-n &lt;ns&gt;</code>, <code>-A</code>/<code>--all-namespaces</code>, <code>-l &lt;selector&gt;</code>, <code>-o yaml</code>/<code>json</code>/<code>wide</code>, <code>-w</code> (watch), <code>--previous</code>, <code>--dry-run=client</code>.</>,
    <><code>-o yaml</code> is the best way to learn the schema of any object - dump it, read it.</>,
    <><code>--dry-run=client -o yaml</code> generates a manifest from imperative args without applying - great for scaffolding YAML.</>,
  ]}
/>

</div>
