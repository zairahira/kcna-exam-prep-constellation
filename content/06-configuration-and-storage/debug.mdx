<DebugPuzzle id="m06-d2" syllabusIds={["K8S-F-05"]}
  prompt={<p>A junior engineer stores a database password in a ConfigMap. A security reviewer flags it. What's the correct fix and why?</p>}
  artifact={null}
  choices={[
    { id: "a", label: "Use a Secret. Secrets get separate RBAC treatment and signal confidentiality - though remember base64 ≠ encryption; enable etcd encryption at rest for real protection." },
    { id: "b", label: "Keep the ConfigMap but add an annotation: confidential=true." },
    { id: "c", label: "Hardcode the password in the container image." },
    { id: "d", label: "ConfigMaps are fine for passwords because they're only readable by the cluster admin." },
  ]}
  correctId="a"
  explanation={<p>Use a Secret. Secrets are base64-encoded (not encrypted) in etcd, but they get separate RBAC, different tooling handling, and audit visibility. For real at-rest encryption, configure etcd encryption. Never hardcode secrets in images - they leak via image scans and registries.</p>}
/>
