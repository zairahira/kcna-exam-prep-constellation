<Checkpoint
  id="m01-q06"
  question="Why was Docker support (dockershim) removed from Kubernetes in version 1.24?"
  choices={[
    { id: "a", label: "Docker containers are incompatible with Kubernetes pods" },
    { id: "b", label: "Docker images use a proprietary format that Kubernetes cannot read" },
    { id: "c", label: "Docker does not implement the CRI — Kubernetes used a built-in shim that added unnecessary complexity" },
    { id: "d", label: "Docker, Inc. asked Kubernetes to remove it" },
  ]}
  correctId="c"
  feedback={<p>Docker does not implement the Container Runtime Interface. Kubernetes maintained a built-in adapter called "dockershim" to translate between the CRI and Docker's API. This added complexity and maintenance burden. Since containerd (the runtime engine inside Docker) does implement CRI and can be used directly, the shim was unnecessary. Removing it simplified Kubernetes without affecting workloads — existing images are OCI-standard and work with any compliant runtime.</p>}
/>
