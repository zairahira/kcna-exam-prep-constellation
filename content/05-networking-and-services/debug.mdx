<DebugPuzzle id="m05-d3" syllabusIds={["K8S-F-03"]}
  prompt={<p>Inside a Pod in namespace <code>analytics</code>, you try to reach a Service named <code>billing</code> in namespace <code>core</code>. Which DNS name works?</p>}
  artifact={null}
  choices={[
    { id: "a", label: "billing" },
    { id: "b", label: "billing.core" },
    { id: "c", label: "billing.core.svc.cluster.local" },
    { id: "d", label: "Both B and C are valid (C is fully qualified; B works because DNS search suffixes get appended)." },
  ]}
  correctId="d"
  explanation={<p>Both work. CoreDNS configures search suffixes so that short names like <code>billing.core</code> resolve via suffix search. The fully-qualified form <code>billing.core.svc.cluster.local</code> is unambiguous. A bare <code>billing</code> would only resolve if the Service is in the <em>same</em> namespace as the caller.</p>}
/>
