  question={<>Which resource is the standard choice for running 5 replicas of a stateless web API with rolling updates?</>}
  choices={[

<Checkpoint id="m04-q3"
  question={<>The ownership chain for a Pod managed by a Deployment is:</>}
  choices={[
    { id: "a", label: "Deployment → Pod" },
    { id: "b", label: "Deployment → ReplicaSet → Pod" },
    { id: "c", label: "ReplicaSet → Deployment → Pod" },
    { id: "d", label: "Pod → Deployment" },
  ]}
  correctId="b"
  feedback={<p>Deployment owns ReplicaSet(s), which own Pods. When you update a Deployment's Pod template, it creates a new ReplicaSet and shifts Pods from old to new.</p>}

<Checkpoint id="m04-q4"
  question={<>Which workload resource gives Pods a <strong>stable identity</strong> and stable storage - appropriate for databases?</>}
  choices={[
    { id: "a", label: "Deployment" },
    { id: "b", label: "StatefulSet" },
    { id: "c", label: "DaemonSet" },
    { id: "d", label: "Job" },
  ]}
  correctId="b"
  feedback={<p>StatefulSet gives each Pod a predictable ordinal name (<code>db-0</code>, <code>db-1</code>) and a persistent PVC that sticks with it across reschedules. Use for stateful workloads where "which replica am I" matters.</p>}
/>

<Checkpoint id="m04-q5"
  question={<>A liveness probe that keeps failing will cause Kubernetes to:</>}
  choices={[
    { id: "a", label: "Remove the Pod from Service endpoints" },
    { id: "b", label: "Restart the container" },
    { id: "c", label: "Evict the Pod from the node" },
    { id: "d", label: "Scale up the Deployment" },
  ]}
  correctId="b"
  feedback={<p>Liveness probe failure → restart the container. Readiness probe failure → remove from Service endpoints (but don&apos;t restart). Different signals, different responses.</p>}
/>

<Checkpoint id="m04-q6"
  question={<>You want a small setup task to run once before your main container starts. Which pattern?</>}
  choices={[
    { id: "a", label: "A sidecar container" },
    { id: "b", label: "An init container" },
    { id: "c", label: "A separate Job" },
    { id: "d", label: "A DaemonSet" },
  ]}
  correctId="b"
  feedback={<p>Init containers run sequentially to completion before the Pod's main containers start. Perfect for setup tasks (database migrations, config rendering, waiting for a dependency).</p>}
/>

<Checkpoint id="m04-q7"

  choices={[
    { id: "a", label: "The node is out of memory" },

/>
