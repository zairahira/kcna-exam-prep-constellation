<Remember
  points={[
    <>A <strong>Pod</strong> is the smallest deployable unit. Contains one or more containers sharing network (same IP, localhost) and storage volumes.</>,
    <>Pods are <strong>ephemeral</strong>. IPs change on restart. Don&apos;t create them directly - use a controller.</>,
    <>Pod lifecycle phases: <code>Pending</code> → <code>Running</code> → <code>Succeeded</code> / <code>Failed</code>. <code>Unknown</code> when the kubelet can&apos;t report status.</>,
    <>Multi-container patterns: <strong>sidecar</strong> (helper alongside main), <strong>init container</strong> (runs to completion before main starts), ambassador, adapter.</>,
  ]}
/>

<Remember
  points={[
    <>A <strong>ReplicaSet</strong> maintains N identical Pods matching a selector.</>,
    <>A <strong>Deployment</strong> manages ReplicaSets to enable rolling updates and rollbacks. Standard choice for stateless apps.</>,
    <>Hierarchy: <strong>Deployment owns ReplicaSet owns Pods</strong>.</>,
    <>Rollout strategies: <code>RollingUpdate</code> (default; gradual swap) and <code>Recreate</code> (kill all, then start new).</>,
    <>Rollout commands: <code>kubectl rollout status</code>, <code>kubectl rollout history</code>, <code>kubectl rollout undo</code>.</>,
  ]}
/>

<Remember
  points={[
    <><strong>StatefulSet</strong>: Pods get stable identity (ordinal name) and stable storage. For databases, anything "which replica am I" matters.</>,
    <><strong>DaemonSet</strong>: one Pod per (matching) node. Log shippers, CNI plugins, monitoring agents.</>,
    <><strong>Job</strong>: run-to-completion workload. Retries on failure. <strong>CronJob</strong>: Job on a schedule (like cron).</>,
  ]}
/>

<Remember
  points={[
    <><strong>Liveness probe</strong>: if failing, container is restarted. "Is it deadlocked?"</>,
    <><strong>Readiness probe</strong>: if failing, Pod is removed from Service endpoints. "Ready to take traffic?"</>,
    <><strong>Startup probe</strong>: used for slow-starting apps; disables liveness/readiness until it succeeds.</>,
    <>Probe types: <strong>HTTP</strong> GET, <strong>TCP</strong> socket, <strong>exec</strong> (run command in container).</>,
    <>Restart policies: <code>Always</code> (default for Deployments), <code>OnFailure</code> (Jobs), <code>Never</code>.</>,
    <><code>CrashLoopBackOff</code> = container keeps exiting. Use <code>kubectl logs --previous</code> to diagnose.</>,
  ]}
/>
