<Remember
  points={[
    <><strong>RBAC</strong> = authorization model. Subjects get permissions via bindings to Roles/ClusterRoles.</>,
    <><strong>Role</strong> (namespaced) vs <strong>ClusterRole</strong> (cluster-wide). <strong>RoleBinding</strong> (namespaced) vs <strong>ClusterRoleBinding</strong> (cluster-wide).</>,
    <>Common verbs: <code>get</code>, <code>list</code>, <code>watch</code>, <code>create</code>, <code>update</code>, <code>patch</code>, <code>delete</code>.</>,
    <>Check permissions: <code>kubectl auth can-i &lt;verb&gt; &lt;resource&gt; --as=&lt;subject&gt;</code>.</>,
  ]}
/>

<Remember
  points={[
    <><strong>ServiceAccount</strong> = workload identity. Namespaced. Default SA exists per namespace.</>,
    <>Tokens auto-mount into Pods at <code>/var/run/secrets/kubernetes.io/serviceaccount/token</code>. Disable with <code>automountServiceAccountToken: false</code>.</>,
    <>ServiceAccounts have no permissions until bound via RBAC.</>,
    <>Cloud workload identity: <strong>IRSA</strong> (AWS), <strong>Workload Identity</strong> (GCP), <strong>Azure AD Workload Identity</strong> - map SA → cloud IAM.</>,
  ]}
/>

<Remember
  points={[
    <><strong>Pod Security Standards</strong> (PSS): three levels - <code>privileged</code>, <code>baseline</code>, <code>restricted</code>.</>,
    <><strong>Pod Security Admission</strong> (PSA) enforces PSS via namespace labels. Modes: <code>enforce</code>, <code>audit</code>, <code>warn</code>.</>,
    <>PSA replaced <strong>PodSecurityPolicy</strong> (PSP), which was removed in v1.25.</>,
    <>Key SecurityContext fields: <code>runAsNonRoot</code>, <code>readOnlyRootFilesystem</code>, <code>allowPrivilegeEscalation</code>, <code>capabilities.drop</code>.</>,
  ]}
/>

<Remember
  points={[
    <>Image security: pull from trusted registries, pin by digest (not tag), scan for CVEs (Trivy, Grype), sign with <strong>cosign</strong> (Sigstore).</>,
    <><strong>Admission controllers</strong> run after authn/authz. <strong>Mutating</strong> modifies requests; <strong>validating</strong> accepts or rejects.</>,
    <>Policy engines: <strong>OPA Gatekeeper</strong> (CNCF Graduated, Rego policies) and <strong>Kyverno</strong> (CNCF Incubating, YAML policies).</>,
