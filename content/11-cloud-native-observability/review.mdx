<Remember
  points={[
    <>Observability's <strong>three pillars</strong>: logs (what happened), metrics (how much/how often), traces (where time went in distributed systems).</>,
    <><strong>Observability</strong> ≠ monitoring. Monitoring = watching known signals. Observability = asking new questions you didn&apos;t think to ask upfront.</>,
  ]}
/>

<Remember
  points={[
    <><strong>Prometheus</strong> (CNCF Graduated) = metrics database. <strong>Pull-based</strong>: scrapes <code>/metrics</code> endpoints.</>,
    <>Metric types: <strong>counter</strong> (monotonic), <strong>gauge</strong> (up/down), <strong>histogram</strong> (distribution), <strong>summary</strong>.</>,
    <><strong>PromQL</strong> queries metrics. Key functions: <code>rate()</code>, <code>sum by ()</code>, <code>histogram_quantile()</code>.</>,
    <><strong>Exporters</strong> translate non-Prometheus sources (node, DBs, Kubernetes state) to Prometheus format. <strong>kube-state-metrics</strong> exposes K8s object state.</>,
    <>Pair with <strong>Alertmanager</strong> (routing alerts) and <strong>Grafana</strong> (dashboards).</>,
  ]}
/>

<Remember
  points={[
    <>Container logs = stdout/stderr. Kubernetes captures them to node disk; <code>kubectl logs</code> reads from there.</>,
    <>Centralize logs for long-term, cross-service queries: <strong>DaemonSet</strong> log collector (most common) or <strong>sidecar</strong> per Pod.</>,
    <>Key tools: <strong>Fluentd</strong> and <strong>Fluent Bit</strong> (CNCF Graduated), <strong>Loki</strong>, Elasticsearch/OpenSearch + Kibana (EFK).</>,
    <><strong>Structured (JSON) logs</strong> beat unstructured for any serious aggregation.</>,
  ]}
/>

<Remember
  points={[
    <><strong>Distributed tracing</strong>: every request gets a trace ID; each hop adds a <strong>span</strong>. Tree of spans = the trace.</>,
    <><strong>Context propagation</strong>: trace ID must be passed downstream (W3C <code>traceparent</code> header). Without it, traces fragment.</>,
    <><strong>OpenTelemetry</strong> (CNCF Incubating) = vendor-neutral standard for metrics, logs, and traces. Merged OpenTracing + OpenCensus.</>,
    <><strong>Jaeger</strong> (CNCF Graduated) is the most common open-source tracing backend.</>,
    <>Use <strong>sampling</strong> (head or tail) to keep volume manageable.</>,
  ]}
/>

<Remember
  points={[
    <>Cost observability is part of the observability story.</>,
    <>Common waste: oversized <code>requests</code>, idle dev envs, orphaned PVs/LoadBalancers.</>,
    <><strong>OpenCost</strong> (CNCF Sandbox) is the open standard; <strong>Kubecost</strong> builds on it.</>,
    <><strong>FinOps</strong> is the operating model - shared financial responsibility across engineering, finance, and product.</>,
  ]}
/>
