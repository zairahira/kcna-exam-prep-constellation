<Remember
  points={[
    <><strong>Kubernetes</strong> (K8s) is an open-source container orchestrator. It automates scheduling, scaling, self-healing, rollouts, and service discovery for containerised workloads.</>,
    <>Kubernetes came from Google&apos;s <strong>Borg</strong> system. Open-sourced in 2014, donated to the <strong>CNCF</strong> in 2015. First CNCF Graduated project.</>,
    <>Kubernetes does <strong>not</strong> build images, provide CI/CD, include monitoring/logging, or manage application code. It is the infrastructure layer.</>,
    <>A <strong>cluster</strong> = control plane + worker nodes. You interact with it through the Kubernetes API (via <code>kubectl</code> or programmatic clients).</>,
  ]}
/>

<Remember
  points={[
    <><strong>kube-apiserver</strong> - the single entry point for all cluster operations. Handles authentication, authorisation, admission control. Only component that talks to <code>etcd</code>.</>,
    <><strong>etcd</strong> - distributed key-value store. The cluster&apos;s source of truth. Uses Raft consensus. Run 3 or 5 members in production. <strong>Only</strong> the API server reads/writes to it.</>,
    <><strong>kube-scheduler</strong> - assigns unscheduled Pods to nodes. Filters infeasible nodes, scores the rest, binds the Pod to the winner. Does not start containers.</>,
    <><strong>kube-controller-manager</strong> - runs reconciliation loops (node controller, ReplicaSet controller, Deployment controller, etc.) that drive actual state toward desired state.</>,
    <><strong>cloud-controller-manager</strong> - optional. Runs cloud-specific controllers (load balancer provisioning, node lifecycle, routes) so cloud logic stays out of core Kubernetes.</>,
  ]}
/>

<Remember
  points={[
    <><strong>kubelet</strong> - node agent. Watches for Pods assigned to its node, manages container lifecycle via the CRI, runs health probes, reports status. Runs as a <code>systemd</code> process, not a container.</>,
    <><strong>kube-proxy</strong> - network component on every node. Maintains iptables/IPVS/nftables rules so Service traffic reaches the right Pods. Does not proxy traffic in userspace (in iptables mode).</>,
    <><strong>Container runtime</strong> - <code>containerd</code> (most common) or <code>CRI-O</code>. Communicates with kubelet via <strong>CRI</strong>. Delegates to <code>runc</code> for low-level container creation.</>,
    <>Nodes register with the control plane automatically. The kubelet sends <strong>heartbeats</strong> via Lease objects. If heartbeats stop, the node controller marks the node <code>NotReady</code>.</>,
  ]}
/>

<Remember
  points={[
    <>All components communicate <strong>through the API server</strong> - no direct component-to-component communication. This enforces centralised auth, audit, and access control.</>,
    <>Components use the <strong>watch mechanism</strong> (long-lived HTTP connections) to receive real-time events from the API server. No polling.</>,
    <><strong>Informers</strong> maintain a local cache of watched resources, reducing API server load. If the watch drops, the informer re-lists and resumes.</>,
    <><strong>Leader election</strong> via Lease objects ensures only one instance of the scheduler and controller manager is active at a time, preventing split-brain in HA setups.</>,
    <>All cluster communication is secured with <strong>mutual TLS (mTLS)</strong>. Each component authenticates with its own certificate.</>,
  ]}
/>
