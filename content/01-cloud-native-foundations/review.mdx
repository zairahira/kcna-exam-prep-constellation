<Remember
  points={[
    <><strong>Cloud native</strong> = building software that takes full advantage of cloud environments. Four pillars: containers, microservices, CI/CD, declarative APIs.</>,
    <>The <strong>CNCF</strong> (Cloud Native Computing Foundation) hosts and governs open-source cloud native projects. It doesn&apos;t build software - the community does.</>,
    <>CNCF project maturity levels: <code>Sandbox</code> → <code>Incubating</code> → <code>Graduated</code>. Graduated = production-ready, widely adopted.</>,
    <>The <strong>CNCF Landscape</strong> (<code>landscape.cncf.io</code>) maps the entire ecosystem. The <strong>Trail Map</strong> is a suggested learning path.</>,
  ]}
/>

<Remember
  points={[
    <>A <strong>container</strong> is a lightweight, isolated environment for running a process. Shares the host kernel (unlike VMs). Starts in ms, sized in MBs.</>,
    <>A <strong>container image</strong> is a read-only template made of layers. Each <code>Dockerfile</code> instruction adds a layer. Layers are cached and shared.</>,
    <>Images are identified as <code>registry/repository:tag</code>. Pin specific tags in production - <code>latest</code> doesn&apos;t auto-update.</>,
    <>A <strong>registry</strong> stores images. Examples: Docker Hub, GitHub Container Registry (ghcr.io), Harbor (CNCF Graduated, self-hosted).</>,
    <>The <strong>OCI</strong> (Open Container Initiative) defines open standards for image format and runtime. Ensures portability across tools.</>,
  ]}
/>

<Remember
  points={[
    <><strong>High-level runtimes</strong>: <code>containerd</code> (most common, CNCF Graduated) and <code>CRI-O</code> (K8s-only, common in OpenShift).</>,
    <><strong>Low-level runtimes</strong>: <code>runc</code> (OCI reference), <code>gVisor</code> (sandboxed), <code>Kata Containers</code> (VM-based isolation).</>,
    <><strong>CRI</strong> (Container Runtime Interface) is the Kubernetes API that lets kubelet talk to any compatible runtime.</>,
    <>Kubernetes removed <strong>dockershim</strong> in v1.24. Docker images still work (they&apos;re OCI-compliant) - Docker-the-runtime is just no longer used by K8s.</>,
  ]}
/>

<Remember
  points={[
    <>A <strong>monolith</strong> is a single deployable unit. Simple to develop, test, debug. Hard to scale parts independently, risky deploys, team coupling.</>,
    <><strong>Microservices</strong> = small, independently deployable services. Independent scaling and deploys. More operational complexity.</>,
    <>Common pattern: <strong>start monolith, decompose later</strong> when team size and product complexity justify it.</>,
    <>Microservice communication: <strong>sync</strong> (REST, gRPC) or <strong>async</strong> (message queues, event streaming). Async reduces coupling but adds complexity.</>,
    <>Kubernetes and microservices are a natural fit but neither requires the other.</>,
  ]}
/>
